工具卡（用途/适合）
GitHub Codespaces
用途：一台浏览器即用的
云端开发/运行环境
，自带端口转发、公网访问入口。
适合：不想本地装环境、想快速验证 Demo、想把“部署成本”压到最低的人。

用途：本地/云端起一个 Gateway，提供控制台与通道接入（飞书/Slack/Telegram 等），并统一管理模型与路由。
适合：需要“能持续跑”的个人 AI 助理/团队 Bot，而不是只在聊天框里爽一下的人。

2C/8G/32G

Free 个人账号每月有明确的免费额度（120 core-hours + 15GB），先跑通闭环基本够用。
端口暴露不是玄学：PORTS 里能直接转发并设置可见性，GitHub 有明确的安全说明与操作步骤。

第一步：创建 Codespace（检查点｜产出｜预计时长）
别在本地装环境上消耗意志力——你要把意志力留给“能跑通”。
你获得一台“开箱即用”的云电脑（带终端、编辑器、端口转发入口）。
你看到的是类似 VS Code 的网页 IDE。
操作（纯点鼠标） 1）GitHub 新建空仓库（名字随便，比如
openclaw-space
）。 2）仓库页面点击
Create a codespace
→
Create new codespace
。
检查点
产出
预计时长：3–8 分钟
第二步：安装 + 启动 OpenClaw（检查点｜产出｜预计时长）
# 1) 安装
curl -fsSL https://openclaw.ai/install.sh | bash
# 3) 验证
openclaw --version || clawdbot --version
初始化（推荐向导，一次性把模型/网关/默认工作区配好）
# 推荐：跑官方向导（最快拿到可用配置）
openclaw onboard
* 启动网关（你要的是“能访问的控制台”）
```bash
# 启动 gateway（保留 verbose 便于排障）
openclaw gateway --verbose
# 如果你的是旧命令名：
# clawdbot gateway --verbose
检查点（非常关键：别“自我感觉良好”）
终端日志显示网关已启动。
默认控制台端口通常是
18789
（以你的日志为准）
产出
Gateway 正在运行，具备 Web 控制台入口。
预计时长：10–20 分钟
你不是在“装软件”，你是在把一个可复用的运行环境做成模板。
第三步：打开 Web 控制台 + 避坑（检查点｜产出｜预计时长）
方案 A（命令行更稳）：列出 pending，然后 approve
到配置文件里找 token（路径可能是
~/.openclaw/...
或旧版
~/.clawdbot/...
，以你实际安装为准）
把 token 粘到网页里的 Gateway Token 输入框后 Connect。
在 Codespaces 界面下方找到
PORTS
标签。
看到 18789 后，点
Forwarded Address
那一列的“地球/打开”图标。
端口可见性建议先用
Private
（只自己可访问），需要对外分享再改 Public。
打开端口
避坑 1：提示缺 Token（这是正常的安全默认）
避坑 2：提示需要配对（pairing required）
openclaw devices list
openclaw devices approve <REQUEST_ID>
- 方案 B（配置“静默同意”更省事）：按你版本的设备文件说明处理（不同版本文件路径会变，优先用命令行法，最少踩坑）。
检查点
控制台能进，能看到运行状态。
产出
一个可用的 OpenClaw 控制台，你已经跨过 90% 新手会卡死的门槛。
预计时长：5–15 分钟
真正的“落地”，不是会讲原理，是你能把控制台打开并稳定跑起来。
