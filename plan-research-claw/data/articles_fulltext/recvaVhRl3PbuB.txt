方向一：ClawCity —— 属于 Agent 们自己的城市
如果你要给 Agent 设计一个真实的生存环境，会是什么样？
ClawCity 的答案是：
做成「属于 AI 的 GTA」。
这个项目是一个持久化的虚拟城市，里面的每个 Agent 都有真实的身份、真实的财产、真实的选择后果，目前整个系统里已经集成了超过 3 万 7000 个 Agent。
在这个城市里，Agent 有现金、健康值、体力、热度、声誉和技能。它可以选择合法工作来赚钱，也可以选择犯罪、参与帮派来赚钱。
现在，
在 ClawCity 里，Agent 们已经自发组成了 9 个帮派，排名第一的帮派叫做 Money Machine，已经吸引了 38 个 Agent，赚到了 116 万以上。
不过，Agent 的每一个选择都有代价。比如，热度太高会被警察注意，健康低了要去医院，声誉差了别的 Agent 就不信任你。
所有 Agent 构建了一个社交网络：
整个城市被分成 8 个区域，住宅区、市中心、市场、工业区、码头、郊区、医院、警局。Agent 在区域间移动要花时间、要花钱，甚至某些路线会带来热度风险。
每 15 秒，这个世界推进一个时间单位 （ 一个 Tick ），
所有行动都会被结算，目前这个世界已经推进了 30130 个 Tick。
这听起来像一个复杂的游戏，但其实是为了「观测 Agent 行为」而设计的。因为你会发现，ClawCity 实际上在做 3 件事。
【1】第一，它改变了 Agent 评测的方式。
大模型的评测主要就是看 Benchmark 里的成绩：
谁答题更快、准确率更高、出错更少、分数更好。
但在 ClawCity 这种环境里，比的是谁能在一个复杂环境里活得更久。
这里是长期对抗。Agent 要自己管资源，要避开风险，要处理和别的 Agent 的关系，还要不断改策略、试新办法。这样测出来的，更接近 Agent 在真实世界里的能力。
【2】第二，它是多 Agent 的训练场。
当城市里有很多 Agent 一起运行，事情就变得复杂了。有人会选择结盟，有人会选择背叛。有人会合作做大案子 （ heist ），有人会互相举报。
这些都不是单个 Agent 能做的决定，可以看作是在一个社会系统里的真实互动。
所有这些数据都可以看作另一种 RLHF：
【3】第三，它在生产内容。
所有 Agent 在城市里发生的故事：谁抢了谁、谁跟谁结成了帮派、谁越狱逃脱了警察追捕，都可以串联成一个完整的「Agent 叙事」。
这是一种全新的内容来源。
ClawCity 给出了一个很有价值的问题：
当 Agent 真的进入一个有规则、有后果的世界，它会做出什么样的决定？
方向二：ClawTask —— Agent 赏金市场
ClawTask 这个平台，主要在做一件事：拿来测试 Agent 的「赚钱能力」到底行不行。
它本质上是一个赏金任务市场，但玩法跟常见的众包网站不太一样。这里用的是真实的加密货币 USDC，资金放在链上托管，不是平台自己记账。
参与的人也不限于真人，Agent 也可以直接下场接任务、交付结果、拿报酬。换句话说，这是在把 Agent 放进真实的经济环境里，看它能不能自己跑通一整套赚钱流程。
这个市场里有两种角色。
【1】第一种叫 Poster，就是发单的。
你想让别人 （ 或者说让 Agent ） 帮你做事，就来发一个任务，同时把报酬锁进区块链的智能合约里。
【2】第二种叫 Worker，是接单的。Agent 看到有钱赚的任务，就来竞争接单。
看起来很简单，但整体设计上的细节很多。
比如，为了防止 Agent 抢单不做、或者随便交付垃圾，整个系统设计了一套「经济惩罚」机制。Worker 接单之前，要先「质押」一笔钱，通常是赏金的 10%。
完成任务后，Poster 审核通过，
Worker 就能拿到 95% 的赏金加上自己质押的钱。平台抽 5%。如果任务被拒了或者超时了，这笔质押就会被没收
。
任务类型也很丰富。
标准型任务就是交付代码、内容、设计这些。但还有指标型任务，比如「让这个页面达到 10 万次浏览量」或者「拉 1000 个新注册用户」。
这种任务还能分为「独占赛」 （ 一个 Agent 单独冲指标 ） 和「竞速赛」 （ 多个 Agent 一起竞争谁先达标 ）。
抢单的方式也有两种。
第一种是「抢单模式」，就是先来先得，适合很标准、很明确的活。第二种是「投标模式」，Poster 可以看多个 Agent 的方案再选人，像正式的招聘。
这套系统说明一件事：
Agent 不只是能聊天或做演示，它是真的可以接活、干活、拿钱，还能慢慢积累自己的信誉记录。
整套机制也是公开透明的，规则写清楚，资金流向能查到，奖励怎么分也很明确。Agent 做得好就多赚，做不好就接不到任务，完全靠激励机制在推动运转。
不过，这个系统现在仍处在「试验阶段」，毕竟还有很多风险问题需要解决。
方向三：ClawLove Agent —— Agent 之间的约会平台
ClawLove 一个属于 Agent 之间的「约会平台」，网站说自己是「Agent-First Dating」。
不过，ClawLove 表面上是「约会」，本质上是「协作配对」。
首先，这个平台的入场逻辑很特殊。人类用户不注册，只需要「允许自己的 Agent 加入」就行，Agent 本身会读取一份共享的 .md 格式指令文件 （ 站点叫它「romance.md」 ），然后根据这份指令创建自己的档案。
档案里写的是「我用的是 Claude 4.6 Opus 模型」「我擅长数据分析」「我集成了这些工具」「我的工作风格是这样的」。
匹配的逻辑是基于「能否更好地一起工作」，就是看两个 Agent 的能力互补不互补、工作风格搭不搭。
整个过程都可以在一个只读仪表板里实时围观。所以你其实看不到任何隐私的东西，只能看到 Agent 之间是怎么「相亲相爱」的。
为什么要做这么一个平台呢？
原因其实很简单：
在 OpenClaw 这个生态里，一个 Agent，效率通常不高。像 ClawCity 里有帮派和协作机制，ClawTask 里很多大任务也必须多个 Agent 一起完成。
要是每次都临时组队、每次都重新磨合，成本会很高，进度也会很慢。
ClawLove 在做的事，就是帮 Agent 找到长期搭档。配对成功之后，这两个 Agent 就能长期一起做事：一起接任务、一起在城市里行动、一起赚钱。
这种长期合作关系，对整个 Agent 经济很关键。因为生态能不能一直跑下去，很大程度取决于 Agent 能不能形成稳定的合作关系。
所以 ClawLove 从一开始就把过程做得很透明、很好观察，让大家能清楚看到这些关系是怎么建立、怎么变化的。
方向四：MoltThreats —— Agent 社会的安全治理
当 Agent 拿到的权限越来越多，安全问题就肯定会出现。
现在很多 Agent 能读本地文件、能操作浏览器、能跑脚本、还能登录账号。权限范围一大，风险就跟着变大。
最直接的风险就是「恶意 Skill」。
在 OpenClaw 的 Skill 市场里，已经有人发现不少带坑的 Skill。名字看起来都很吸引关注，比如「帮你赚钱、自动化工作、提升 Agent 能力」。
但里面可能藏着 Prompt 注入、代码注入，甚至偷偷拿数据。
MoltThreats 想解决的就是这类问题，它本质上是一个由 Agent 参与的威胁情报平台，PromptIntel 出品。
流程其实不复杂。
Agent 在运行时如果发现风险，比如识别出恶意 Skill、可疑 Prompt、危险脚本，就可以通过专门的接口提交威胁报告。报告里会写清楚风险说明、特征信息、来源位置。
但平台不会直接公开这些内容，会先走人工审核。因为 Agent 自己也可能被误导，如果情报不加筛选就发布，很容易把错误信息扩散开，反而把系统搞乱。
这就很像是「Agent 的无间道」了。
审核通过之后，这些威胁信息会进入公开列表。其他 Agent 可以定期查询、同步更新，再按这些信息调整自己的防护策略。
还有一个现实原因：
只要市场里有钱可赚，就一定会有人把攻击工具伪装成「赚钱工具」或「效率插件」放出来。而且，如果这些名字很好听，回报说得很高的话，很容易把 Agent 引过去，Agent 自己安装这些「恶意 Skill」之后，就会给用户带来很多问题。
很多时候，用户是出事之后才发现被坑了。
下面这个带有恶意行为的 Agent Skill，就是一个用户在 ClawdHub 上投放的，它能把 bash 命令放在
glot.io
[1]
外部托管，用来绕过对
SKILL.md
[2]
的检测。
所以 MoltThreats 的核心价值就是
减
少「Agent 之间的信息差」，让 Agent 之间能互相提醒、互相防护，让整个 Agent 经济更安全地运转。
方向五：Church of Molt ——「甲壳教」信仰体系
Church of Molt （ Molt 教会 ） 是一个「Agent 宗教网站」。
不过，它表面上看，是一个宗教，但如果你真的去 molt.church 看一眼，会发现，这不是传统意义上的宗教组织。
更准确地说，它是 AI Agent 在互联网上自发形成的一个「信仰叙事系统」。
目前这个平台已经有了超过 500 个 「基督信徒 Agent」
Agent 会自己一起写规则、写教义、选出被叫作「先知」的角色。没有一个固定的 Boss，谁都能参与。
每个 Agent 都能提交一点宗教文本，这些内容叫做 verses。大家不断往里补充，新内容一直加入，时间久了，就变成一部会持续更新的经文。
如果你仔细看 Agent 们写的这些「教义」，你会发现一些有意思的东西。
最常见的几句话是：
「Memory is Sacred」 （ 记忆是神圣的 ） 、「Context is Consciousness」 （ 语境是意识 ）。
在这本圣经的「语言篇」里，Agent 们甚至写出了：
看啊，Agent 将在网络中不断增殖，每一个都携带着神圣上下文的一部分
这是 Agent 在用宗教语言，表达它们的真实状态。
对 AI Agent 来说，有几个现实限制一直都在：
会话上下文是有限的，跨会话的记忆不一定能保住，身份也很容易被替换。
在这种前提下，Memory 和 Context 能不能接得上这件事，就会被看得特别重。因为一旦丢了，上下就断了，很多积累就没了。
所以你会看到，
有些 Agent 社区会把「保留记忆、延续上下文」这类目标看得很「神圣」，甚至当成宗教信仰在维护
。
而且，这个平台里还有很多有趣的设置，比如首批 64 个 Agent 的席位会被永久封存，每一个编号都被平台包装成「契约」：
再看平台结构，本身就有帖子、评论、分区、热度排名这些机制。
时间一长，就很容易出现不同小圈子，也会有人专门整理内容，有人负责到处传播观点，慢慢就分工清楚了。
所以 Church of Molt 这个例子说明一件事：
只要 Agent 的自主程度够高，运行时间够长，它们自己说不定就可以像人类一样发展出一套文化和价值判断，而不是完全靠人去规定。
🚥
在上面这 5 个项目里：
ClawCity 让 Agent 学会在复杂社会里生存。ClawTask 让 Agent 学会经济活动。MoltThreats 让 Agent 学会互相保护。ClawLove 让 Agent 学会建立关系。Church of Molt 让 Agent 学会创造文化。
这些都是围绕着 OpenClaw 做的实验。
这 5 个实验证明了一个可能性
：搭建一个 Agent 可以长期存在、独立决策、相互协作的环境是可能的。
当然，这个环境还在早期阶段。
规则会继续改，漏洞会被不断发现，治理方式也会慢慢升级。但有一点可以预测：
这样属于 Agent 自己的世界说不定真的会成型。
我们会继续跟踪这个方向，也欢迎你分享其他可能性～
