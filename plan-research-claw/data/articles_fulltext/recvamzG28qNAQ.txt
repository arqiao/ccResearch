叶天奇觉得，ClawdBot（
现改名 Openclaw、曾改名 Moltbot，后文继续用 ClawdBot 指代
） 火得莫名其妙。在他看来，这就是一个胶水沾了很多东西的傻瓜包，像早年安卓时代，大家都会安装很多预装包一样。
不过他也不得不承认，这个在中美两国被病毒式传播的、新的 AI 开源产品，其最大的价值在于降低了与 Agent 的交互门槛。
但类似的解决方案早就有了，甚至几个月前，叶天奇推翻过这种做法。当时他觉得做 “Agent Layer”（Agent 层）最终其实是躺在 “大厂的火车轨道” 上；现在 ClawdBot 所做的优化交互、记忆工程，都属于短时间的技术投机，长期一定是大厂做得更好，就比如 Anthropic
对 ClaudeCode 的持续优化。
放弃做 Agent 层，叶天奇认为最需要的是给 Agent 提供一个 Runtime（运行时间），给它提供一个物理的设备。
因此，创业者叶天奇做了一个内置 ClaudeCode 的微型盒子，通过自己开发的应用，在手机上就可以直接与这个 Agent 交互。
粗暴地理解，他做了一个硬件版的 ClawdBot，只不过你不是通过 WhatsApp 与它交互，而是通过它自带的应用程序来操控它。
叶天奇 1997 年出生，毕业于美国普渡大学，主修机械工程和控制理论；大学期间，他与现在的联合创始人曾一起参加过大疆的机甲大师比赛，这段经历为他埋下了硬件创业的种子。
毕业后他加入了高通，研究机器学习算法以及将算法部署到骁龙处理器上；他也参与过自动驾驶数据管道搭建，积累了从机械到神经网络的全栈能力。
工作两年后，他感到成长空间受限，想要自己做点什么；尝试过 AI 日交易、AI 心理医生、AI 女友后，他意识到，Idea Founder Fit（创业者与点子匹配度）非常重要，最终又回到了自己禀赋最擅长的硬件领域。
叶天奇做的微型盒子（或者叫 Agent 电脑）名为 Pamir AI，这是一个售价 250 美元、计算器大小的硬件，它能 24 小时保持在线，接管那些重复性工作，甚至能通过物理接口，直接介入物理世界。
五源资本的合伙人孟醒形容它是一个非常有技术前瞻性的产品；目前这个仅有 4 名员工（包括创始人在内）的创业项目估值 2500 万美元。
Pamir 用户买家秀，下为
Mac mini
上面是 Pamir AI
Agent 为什么需要一台实体电脑？
很多配置 ClawdBot 的用户最终都把它运行在一个旧的电脑里；这个 24 小时工作的 Agent 也带动了 Mac mini 一轮热卖。大家终于意识到 Agent 需要一台自己独立的 “电脑”。
叶天奇第一次产生当前的产品灵感，是他正尝试用 Agent 自动化写一段代码时，当时他盯着自己的电脑屏幕发呆了两分钟，突然觉得无所事事。
“其实我想用电脑去看点别的东西，但是如果切屏幕的话，我就看不到他在干什么了。” 他意识到，自己在和 Agent 共用一台电脑；而这个 Agent 未来一定会越来越聪明，帮他干越来越多的活，成为他真正的 co-worker。那换句话说，他在跟一个同事共用一台电脑！
“当时我就觉得
Agent
一定需要一台电脑”，从所谓的第一性原理推导，他推导出了 Agent 电脑的三个刚需：
要有物理独立性，与运行在云端或笔记本插件里的 Agent 不同，作为一台独立的终端，它需要能独立在后台 7x24 小时干活。
要提供硬件接口，这意味着 Agent 可以不再被困在浏览器里，可以 “感知” 并操作其他物理硬件 。
要做隐私隔离，既然 Agent 未来要替用户处理私人文件，那么物理隔离，就可以通过硬件手段进行加密和版本回滚 。
叶天奇在芯片公司工作过，他的联创之前在微软 做过产品，他形容俩人的动手能力，“火箭我们也能造出来”，于是第一代产品由他和联创在车库 “手搓” 了出来。
他们结合了 3D 打印的外壳和类似树莓派的模块，做出来一台本质是微型 Linux 操作系统的电脑，95% 的任务可以在本地设备上执行。
叶天奇形容 Pamir AI 跟让 Agent 跑在云端虚拟电脑——比如 Manus——的差别：前者需要持续的费用支出，后者则可以在本地 7x24 小时持续运行，且保存所有的状态和记忆，类似于 “租房” 与 “买房”。
而与配置在
Mac mini
上 ClawdBot 的差别，则在于它的物理连接能力。
通过 USB 接口连接打印机、传感器等外部设备，Pamir AI 可以重写驱动或烧录代码来改造这些硬件（例如让打印机画图、制作智能番茄钟），他形容这是 “热拔插” 的能力。
五源资本的合伙人孟醒就曾被这个能力惊艳。
当时孟醒一直想找一个给硬件进行嵌入式编程（Vibe Coding for Embedded System）的项目，叶天奇的产品不仅符合他心目中的设想，而且不用微调，程序就可以跑起来。
当时叶天奇用这个产品向他展示了两种效果：一是控制灯带，改写了 LED 灯的颜色；二是 Hack 了一个打印机，改变了原本只能打标签的模式，让它可以跨标签打印复杂的图形（如五角星）。这种 “即插即用” 的硬件改写能力，让孟醒当场就决定投资。
目前 Pamir AI 的用户因此也囊括了几类。
一类是开发者，帮他们 24 小时后台写代码；一类是电子发烧友与创作者，让 Agent 帮他们给硬件写运行的程序；还有一类是知识工作者，他们把设备当作 “智能硬盘”，不用拖拽文件到网页对话框，本地就能让 Agent 处理相应工作。
终极定位：一个 “无情的工作机器”
虽然投资人最初是被 Pamir AI“Hack 物理世界” 的能力所吸引；不过，Hack 硬件似乎只是叶天奇在做这个产品中，不设限的做法所衍生的一个功能。
他承认，目前的电子发烧友给了他们很大的信心，也帮助公司迭代了产品。但基于对一个更重要赛道的判断，叶天奇更大的梦想仍是打造一台专属于 Agent 的 “独立电脑”。他希望定义下一代的计算设备形态，最终取代现在的笔记本电脑，辐射到每个工作的人，而不仅仅是程序员或者硬件爱好者。
叶天奇认为，如果电脑只是用来工作的设备，它就应该是这样：与一个前台的设备进行配合（
比如手机、iPad 或者是 AR 眼镜
），其中前端显示设备用来做一些视觉的决策；后台的 Pamir 则主要执行代码运行、数据处理等任务。
他对 Pamir AI 终极的设想是一个 “无情的工作机器”。
为了实现这个目标，也为了更加聚焦，接下来他计划对产品进行功能和需求的收敛，比如砍掉一些为 Hack 服务的功能（例如传感器的接口），让产品更聚焦于电脑的属性。
这一点曾经在 ClawdBot 火之前，叶天奇和投资人孟醒并没有达成共识。
孟醒认为 “Agent 电脑” 虽然是一个合理的赛道，但进行嵌入式硬件编程的想象空间远比在网页上写代码要大，也更有趣。
ClawdBot 的火爆验证了这个需求之大，也让孟醒最终认可了叶天奇的路径，完成了一轮逻辑自洽。
他说，“硬件 Hack” 本质上是一个更长尾的场景，Agent 电脑更加高频，无论从哪个点切入，最终都需要覆盖全场景、殊途同归。
孟醒很看好这种低门槛、普惠的 Agent 硬件产品的空间，因为它能覆盖从最极客到最普通的大众市场。
叶天奇自己也挺有信心，Pamir AI 还在手工制作阶段，每月销量约 200 台，主要通过黑客松的口碑传播和 Instagram 广告获客，但转化率很高，“打广告 15 分钟就开始有人下单”。
目前团队正计划 2026 年 6 月份量产下一代产品，商业模式仍以以硬件销售为主，未来通过深圳供应链的量产能力，能达到 50% 的毛利。
他认为，现阶段产品力已经被验证，接下来的销售无非就是推广，“我只要拧一个旋钮就行了”。
Pamir 目前核心团队只有 4 人，2 个创始人，2 个员工。过去一段时间，他们累计拿了 260 万美元的投资，也从孵化器搬到了车库。
从一个技术极客到一个 Founder、一个创业公司的 CEO，叶天奇和大部分的创业者一样，都经历过低谷，
目前似乎正走上了一个上坡
。
他觉得在这个过程中，最核心的一个教训是要在一个有前景的方向上 “死磕”，坚持长期主义才会迎来曙光，“创业成功非常非常的难，某种意义上，你要下一个长远的赌注。”
以下是筛选过的部分问答，与正文互为补充。
晚点
：我最开始粗暴地理解，觉得你们的产品有点像硬件版的 Clawdbot，你觉得相似的地方是什么，完全不一样的地方是什么？
叶天奇
：思想上都是都是说我们要给
Agent
配一台电脑。他们是相当于他们纯做的软件，我们的话是围绕硬件层和环境层去做。
其实他们这一整套东西，我们内部也做过。后来结论是，我统称他们这种属于 Agent layer，我们认为
Agent
层不是我们的战场。因为
Agent
层的竞争很激烈，而且会有无数的人去打磨它。
我们应该，在早期是把自己的短板抬得很高，就是做一台非常非常好用的
Agent
电脑，然后这个它的中间的
Agent
层是可以热拔插的。因为接下来
Agent
的生态肯定会是百花齐放的。
晚点
：Clawdbot 对在交互上的一些简化，做长记忆这个事儿，会对你们有什么启发吗？
叶天奇
：我们看过他们的代码，发现很多用的方式是我们想过一模一样的解决方式。其实做
Agent
这这一套，你只要走在前沿，大家想的就是会非常收敛。
但当时我们就是把这一套摒弃了，因为我们觉得不符合第一性原理。这些都是像 Anthropic 这种大厂会触及的东西，大厂它是控制着模型的，迭代的这个过程中 solution 一定会更全面我们不想躺在这个大厂的这个火车轨道上跟他们做同样的事情。
所以堆工程我们认为不是一个明智选择。我们的 vision 需要有一个六个月的前瞻性思考，我们要做有技术积累、有复利价值的东西。
晚点
：那你们在记忆上是怎么做的？
叶天奇
：每一家有每一家的做法，他们（Clawdbot）的做法是非常非常工程化的，把大量信息存起来，然后 token 消耗率也很高，最终再从这个 database（数据库） 里抽取这个这个记忆，但是我们认为这个方做法不是很优美。
我们希望更像人类工作一样记笔记。我可能某个项目，这里记一点那里记一点，你工作忘了一些事情，你要么是去飞书里搜一下，要么是去邮箱里去搜一下。记忆是很零散化的，我们的
Agent
也是在在默默去零散化帮你记笔记，然后通过这个方式去持久化你
Agent
的能力，相当于你用了一段时间之后，他记了非常非常多的碎片笔记。
发现你犯错或者是
Agent
犯错的时候，我就记下来，这个变成错题本。
晚点
：什么是技术积累的和有复利价值的东西？
叶天奇
：比如我们刚开始做过端测模型去卖，但后来发现用户的快乐来源于他创造过程本身，而不是说这个东西在不在端侧，它并没有复利。因为它这个迭代和和更新速度那么快。
我们认为复利的点，是围绕增长最快的东西去做，即使我们今天什么都没做，什么进展都没有，公司也在跟着成长。
我们把这个硬件就改为跑
Agent
，是因为我们认为
Agent
生态是一个高速增长的生态，即使我们今天什么都没做，我们公司也在增长。
晚点
：技术上，你们是怎么定义 Agent 的？
叶天奇
：就是大模型一个 while loop（循环），大模型在循环对话的过程中去用一些工具。之前大模型没有这个
Agent
的能力，因为他训练的数据都是问答场景，后来训练的数据增加了多轮行动，模型就具备了这样的这样的能力。
一个大模型，配上 while loop，加上一个工具箱，就是 Agent。
晚点
：通过 whatsapp 这种通讯工具交互似乎也是 Clawdbot 火的原因之一，这种对话是最好的形态吗？
叶天奇
：这就是为什么我说他们的受众人群可能更是知识工作者。你像我们做软件开发的，通过 whatsapp 去跟他交互，这个太痛苦了。
我进行软件开发的时候，要看的东西很多，你不可能只给我一个绿泡泡。我想 debug 下这个文件、那个文件，我想看一下这个 log 、那个 log 怎么办？接收不了，还是很麻烦的。
就算是知识工作者，这个这种交互形式也会碰到瓶颈。因为如果你这时候你突然需要浏览一个文件怎么办？你不能每次说我浏览一个文件，我都要给他发个信息，说你帮我把这个文件。通过短信的形式给我展示出来了，对吧？
我们也还在寻找新的交互形式，比如说你的经常性的聊天变成一些笔记，或者是可搜索的这个知识积累。然后这些知识积累怎么给用户呈现。
晚点
：为什么你们的产品想做的是未来的电脑，而不是手机？
叶天奇
：我们想慢慢取代电脑，其实我们今天公司内部就已经在做这个事情，就是我们上班不带笔记本带 ipad。
大部分工作在我们这个这个小盒子里就可以做了，带电脑也没有什么意义，这是我们最终想达到这样的状态，但是我们不认为手机会被取代，因为最终你还是需要一个一个 interface（界面） 去调用 Agent。不是手机，就是未来的眼镜。
