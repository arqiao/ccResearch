一、准备工作：选择你的AI“大脑”
OpenClaw支持多种模型接入，推荐以下高性价比方案：
1. DeepSeek-V3.1
推荐通过 SiliconFlow（deepseek官网也可以） 接入，可获得 164K 超长上下文窗口，适合处理大型代码库。
• 获取 Key：在 SiliconFlow 官网 注册并获取 API Key。
• 配置步骤：
1. 运行 openclaw onboard。
2. 选择 OpenAI Compatible 模式。
3. 设置 Base URL 为 https://api.siliconflow.cn/v1。
4. 填入密钥，模型 ID 填写 deepseek-ai/DeepSeek-V3.1。
2. Kimi 2.5 (审美与前端任务首选)
Kimi 2.5 在设计榜单表现优异，适合处理网页编写等复杂任务。
• 获取 Key：登录 Kimi 开放平台 https://www.kimi.com/code/console , 在控制台创建 API Key。
• 配置步骤：
1. 运行 openclaw onboard。
2. 在 Model/auth Provider 中选择 Kimi Code API Key。
3. 粘贴密钥并设为默认模型。
3、 本地开源模型接入 (100% 隐私保护)
如果你希望数据完全不经过云端，可以使用 Ollama 或 LM Studio 运行本地模型。适合对数据隐私要求极高的用户。
二、飞书接入详细教程（附避坑指南）
步骤1：安装飞书插件
在终端执行以下命令：
https://github.com/m1heng/clawdbot-feishu
复制下面指令，粘贴到终端回车。
openclaw plugins install @m1heng-clawd/feishu
安装后会显示如下界面。
安装完插件后，会重启一次网关以加载新插件
步骤2：创建飞书应用
你需要为你的OpenClaw智能体创建一个“身份”。
1. 创建应用： 访问 飞书开放平台
https://open.feishu.cn/app?lang=zh-CN
，点击「创建企业自建应用」，填写名称（如“我的AI助手”）和描述。
2. 获取凭据： 在「凭据与基础信息」页面，复制并保存 App ID 和 App Secret。
3. 启用机器人： 在左侧导航栏点击「添加应用能力」，选择「机器人」，点击添加。
步骤3：将凭据填入 OpenClaw
回到你的电脑终端，将刚才获取的 App 信息告知 OpenClaw：
openclaw config set channels.feishu.appId "你的App ID"
openclaw config set channels.feishu.appSecret "你的App Secret"
openclaw config set channels.feishu.enabled true
openclaw config set channels.feishu.connectionMode websocket  # 强烈推荐 WebSocket 模式，无需公网 IP
设置完成后再次运行
openclaw gateway restart
重启网关。
步骤4：开通飞书权限与事件（关键！）
这一步是机器人能否正常沟通的关键。
1. 批量开通权限： 在飞书后台「权限管理」-
「开通权限」
中，确保勾选以下权限：
◦ contact:user.base:readonly（获取用户信息）
◦ im:message（收发消息）
◦ im:message.p2p_msg:readonly（读取私聊消息）
◦ im:message.group_at_msg:readonly（读取群聊 @ 消息）
◦ im:message:send_as_bot（以机器人身份发送消息）
◦ im:resource（上传图片或文件）
2. 配置事件订阅： 在「事件与回调」页面：
◦ 订阅方式：选择 “长连接” (WebSocket)。
◦ 添加事件：搜索并添加 im.message.receive_v1（必需，用于接收消息）。
下面几个也可以搜索加
• im.message.message_read_v1
• im.chat.member.bot.added_v1
• im.chat.member.bot.deleted_v1
步骤5：发布版本并开始聊天
1. 版本发布： 在飞书后台「版本管理与发布」页面，点击「新建版本」，填入版本号（如 1.0.0），保存并发布。
安全配对 (Pairing)：
◦ 在飞书搜到你的机器人，发送“Hello”。
◦ 重要提示：此时机器人不会回复。你需要回到服务器终端，会看到一个 6 位配对码1213。     ◦ 在终端运行：openclaw pairing approve feishu <6位代码>
3. 开始使用：
配对成功后，
你就可以像发微信一样给它下指令
了。例如：“帮我把下载文件夹里最新的 PDF 发给我”。这篇文章篇幅太长，这里就不截图了。
二、 接入钉钉
钉钉用户的配置逻辑非常相似，核心在于利用 Stream 模式 穿透内网。
1. 创建机器人
登录 钉钉开放平台 -> 「应用开发」 -> 「企业内部开发」 -> 「机器人」。
创建应用，记录下 AppKey (Client ID) 和 AppSecret。
2. 权限全开
在后台“权限管理”中，必须开通：
✅ im:message (机器人收发消息)
✅ contact:user.base:readonly (读取用户)
✅ im:resource (文件上传)
3. 终端一键配置
OpenClaw 提供了交互式菜单，非常方便：
终端输入：openclaw channels add
键盘上下键选择 DingTalk，回车。
按提示粘贴你的 AppKey 和 AppSecret。
关键设置：在“开发配置” -> “消息推送”中，务必选择 “Stream 模式” (长连接)。 (这同样意味着：你不需要公网 IP 也能跑通！)
功能特性
✅ Stream 模式 — WebSocket 长连接，无需公网 IP 或 Webhook
✅ 私聊支持 — 直接与机器人对话
✅ 群聊支持 — 在群里 @机器人
✅ 多种消息类型 — 文本、图片、语音（自带识别）、视频、文件
✅ Markdown 回复 — 支持富文本格式回复
✅ 互动卡片 — 支持流式更新，适用于 AI 实时输出
✅ 完整 AI 对话 — 接入 Clawdbot 消息处理管道

接入

如果你主要在企业微信办公，想利用其强大的“自建应用”能力接入 OpenClaw，流程如下：
与飞书/钉钉不同，企业微信的 API 回调通常要求公网可访问。

或者，建议直接将 OpenClaw 部署在云服务器（VPS）上，这就省去了穿透的麻烦。
🛠️ 详细配置步骤
创建“数字员工”
登录 企业微信管理后台，进入「应用管理」 -> 「应用」 -> 「自建」。
点击「创建应用」，起个名字（如：OpenClaw 助手），传个帅气的头像。
关键设置：可见范围务必包含你自己和需要使用机器人的同事。
获取“通关密码”
在应用详情页，复制并保存 AgentId 和 Secret。
进入「接收消息」 -> 点击「设置 API 接收」。
随机生成并保存 Token 和 EncodingAESKey。
OpenClaw 终端配置
回到你的服务器终端，输入指令：
安装插件（如需）：
openclaw plugins install @openclaw/wechat-work
(注：具体包名可能随版本更新，请以官方文档为准)
设置连接模式（二选一）：
方案 A (推荐尝试)：如果新版插件支持 WebSocket，优先使用，可规避公网问题。
方案 B (常规 Webhook)：配置回调地址。 (再次提醒：此模式下，填入企微后台的 URL 必须是公网 IP 或已备案域名！)

五、避坑锦囊
连接失败：如果飞书显示“未建立长连接”，通常是因为你没在终端执行 openclaw gateway restart，或者 App ID 填错。
权限全开： 在钉钉或企微后台，务必勾选 “接收消息” (im.message.receive) 权限，否则 AI 虽然配置好了也听不见你说话。
成本预警：OpenClaw 的 Token 消耗非常快（可能两天烧掉 3000 万 Token），建议接入性价比更高的 DeepSeek V3.1 或 Kimi 2.5 模型。
安全风险
：OpenClaw 拥有极高的系统权限（Root Access），请务必开启配对模式（Pairing Mode），并仅在信任的设备上使用，
也不要随便把它拉进陌生群聊。
长连接优先： 务必让 AI 设置为 websocket 或 Stream 模式，这样你就不需要折腾复杂的内网穿透和公网 IP 了。
