为什么你需要一个 OpenClaw(Clawdbot)？
在开始动手之前，咱们先聊聊底层逻辑。 很多人装了各种 AI 网页端，但总觉得差点意思。为什么？ 因为它们被锁在浏览器里。
OpenClaw 的核心魅力在于：它把 AI 变成了一个可以被聊天软件（比如飞书、TG）调度，且拥有无限技能（Skills）的数字分身。你不需要对着网页打字，你只需要在飞书里直接和它对话，它就能调动你预设的Skills，帮你批处理图片、剪视频、爬数据、写代码。这才是真正的生产力工具，而不是那个只会一本正经胡说八道的聊天框。

01.用 AI 编程工具，五分钟完成极速安装openclaw
很多新手一听到安装项目就头大，觉得要配置什么 Python 环境、Docker 之类的。哥们儿，现在是 2026 年了，我们要学会用 AI 来搞定 AI。
首先要确保你又一个AI编程工具，cursor、antigravity、kiro、windsurf、codex、claude code或者国产的codebuddy、trae等，都行，你能用上哪个用哪个。推荐你想办法搞到一个antigravity，这个是我用过最好用的。
接下来安装openclaw。
第一步：去 GitHub 寻找母体
直接在 GitHub 搜索 openclaw，在地址栏把项目地址复制下来：
https://github.com/openclaw/openclaw
第二步：让 AI 编程工具帮你搞定一切
在本地新建一个文件夹，然后用AI编程工具打开，接着把刚才复制的GitHub项目地址发到AI编程工具的对话框里，然后加上一句话：
“帮我把这个项目https://github.com/openclaw/openclaw
安装到我的本地，确保所有依赖和环境都配置正确，并且需要继续指引我完成后续的配置。”
02. 获取英伟达的免费 API
刚才说了，为了避开那些几千块的学习机坑，我们要找最顶级的免费算力。英伟达目前为了推广自家的开发者生态，简直是大放水，
免费的API额度足足的，每分钟能调用40次，里面的Minimax M2.1模型非常适合来做openclaw这种AI Agent项目。
地址：
build.nvidia.com
登录后点右上角个人中心-API Keys，进去后点击Generate API Key，然后设置个名字，设置有效期（根据你的需求随便设），点击Generate Key生成你的免费API Key。然后把API Key找个地方存好。
除了 Key，你还需要记下它的 API URL（也就是 Endpoint）和 模型名称。
Api url：
https://integrate.api.nvidia.com/v1
Api key：这里填你的API Key
模型名称：minimaxai/minimax-m2.1
回到AI编程工具的对话框，这个时候大概已经安装好了，AI会问你想用什么模型、并且向你要API Key了，把上面的3个组合一起粘进去。
这个时候AI就会帮你帮API key配置好，之后就能用openclaw了。
里面有内置的skills，你也能自己安装，安装方法上篇文章有说👉

03. 硬核接入飞书：把 AI 装进你的口袋
很多教程写到安装成功就结束了，但那只是完成了10%。真正的神级操作，是把 OpenClaw 接入聊天工具，但是国外的Whatsapp、discord或者TG都不太适合国内用，所以我们用国产的工具：飞书。
想象一下，你走在路上，突然想起一个复杂任务，掏出手机给飞书机器人发个消息，它就开始在远程你自己电脑的后台帮你干活了，是不是很爽。
在开始给openclaw配置飞书插件之前，你先要去飞书开放平台申请个账号，然后在你自己的手机端安装一下APP方便随时用。
飞书开放平台：
https://open.feishu.cn/app?lang=zh-CN
第一步：安装飞书插件。
在github上搜“clawdbot-feishu”，这是个给openclaw配置飞书插件的开源项目。
项目地址：
https://github.com/m1heng/clawdbot-feishu
复制这个地址，仍旧是丢给AI编程工具的对话框，让它帮你安装这个插件，
同时需要让AI帮你安装飞书服务端SDK,这一步很重要，否则机器人会没法用.
服务端SDK文档：
https://open.feishu.cn/document/server-docs/server-side-sdk
综合起来就是告诉AI：
根据
https://github.com/m1heng/clawdbot-feishu
这个项目地址帮我安装飞书插件，同时根据这个SDK文档
https://open.feishu.cn/document/server-docs/server-side-sdk
帮我配置服务端SDK
第二步：飞书开放平台配置（这是最容易踩坑的地方）。
1、打开 飞书开放平台，点击右上角开发者后台，进去创建一个“企业自建应用”。
2、添加机器人： 在“添加应用能力”里找到机器人，点击开启。
3、打开基础信息-凭证与基础信息，获取凭证： 复制 App ID 和 App Secret，
然后把App ID 和 App Secret发给到AI编程工具对话框，告诉AI：
根据下面信息帮我继续配置，配置完重启网关：
App ID：XXXXX（你的API ID）
App Secret:XXXXX(你的API KEY)
set channels.feishu.enabled true
4、回到飞书【权限管理】页面，把下面权限都开通权限申请： 这一步千万不能漏，去权限管理里搜索
·im:message
·contact:user.base:readonly
·im:message
·im:message.p2p_msg:readonly （需要bot能力）
·im:message.group_at_msg:readonly （需要bot能力）
·im:message:send_as_bot
·im:resource (上传图片或文件资源)
5、点击【事件与回调】订阅事件：重点来了，很多人这里配置会出错。
事件配置和回调配置中，订阅方式都选“长链接”，然后点击保存。
保存完以后，在事件配置中点“添加事件”，把下面这几个加上：
·im.message.receive_v1（必需）
·im.message.message_read_v1
· im.chat.member.bot.added_v1
· im.chat.member.bot.deleted_v1
配置完成后，在【版本管理与发布】页面创建版本并发布。
这时打开飞书，就能找到应用机器人。然后就可以愉快地发指令了。
