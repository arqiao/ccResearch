今日严寒，适合居家码字。
简单讲讲最近持续火爆的
Clawdbot / Moltbot / OpenClaw吧，命名是 IT 项目难度最大部分。
本质
首先 OpenClaw 是个大模型Agent，如果还没有弄清楚这个概念，可参考
图解LLM-Agent大模型智能体
。
因此具备最重要的两个特点：
1.使用自然语言提示（prompt）大模型推理，
2.有记忆（memory），会规划（planning），能用工具（tools）
技术背景的小伙伴还可以这么看：
1. 自然语言编程，大模型作为自然语言编译器，事实上
OpenClaw 就是对一个微内核 code-agent的封装，
2. 存储交互历史，编制流程或workflow，调用工具的API
OpenClaw 就是这样一个实现，其实Manus也是，最大区别仅关乎部署在哪，工具能干啥。
架构
基于
OpenClaw 官方文档
https://openclaw.ai/，笔者梳理其架构如下：
用户交互界面：Instant Message 各种即时消息渠道
网关：会话管理与路由，控制队列等
智能体执行器：负责模型选型，与提示词组装（来自 记忆+流程和工具）
智能体循环：生命周期，接输入，大模型交互，跑流程，组织输出
Agent 领域不断发明新词，再次表明 IT 行业是第二个fashion show时装行业。
Memory 是对历史对话所有提示词的结构化选取方法，无他；
Langchain, Workflow, Skills 则是在逐层结构化抽象流程与工具的提示词。
技术小伙伴可以持这个视角：
1.
OpenClaw 能将自然语言翻译成命令行调用，
2. 语义理解还是发生在大模型，
3. Agent 仅负责处理结构化组织输入输出
局限
OpenClaw
可以在多个可选大模型中灵活切换，却没有增加智能；
专家提供的流程工具提示（Skills）易用易扩展，但
Planning无长进，类似
Manus 四种适应都没有提升
。
肉眼可见的几个潜在问题：
1. 大模型的幻觉还在，从内容生成，层层传播放大到现实任务执行，
2. 精准控制难，60-70分的demo容易，90分生产环境要求难于上青天，
3. 安全与隐私风险，薅羊毛点奶茶可以，把root权限和银行App密码给它，得勇敢无畏+不差钱。
