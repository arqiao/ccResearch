在家搭建你的专属AI助理：OpenClaw本地化部署全记录
Sienna-banner-geek.png
前言
作为一名总爱折腾新工具的设计师，我最近迷上了能自动化处理各种任务的AI助手。但说实话，把AI完全交给云端总有点不放心，数据隐私、使用成本都是问题。所以，我决定亲手把OpenClaw——这个功能强大的开源AI助手——请到家里的服务器上，打造一个完全受我控制的“数字管家”。
今天这篇笔记，就记录下我从零开始，在PVE虚拟环境中部署OpenClaw的全过程。这不是一篇官方教程，纯粹是我的个人实践分享，希望能给同样爱折腾的你一些参考。
我的核心目标有三个：
「本地化运行」
：核心服务跑在我自己的机器上（注意，不是本地大模型，是服务本身）。
「安全隔离」
：通过虚拟机实现环境隔离，可控又安心。
「全家共享」
：接入Discord，让家人也能一起“使唤”这个AI助理。
准备好了吗？我们开始吧！
前期准备
❝
硬件上，其实只需要一台能24小时运行的机器就行。如果想跑本地大模型，那才需要更强大的配置。
❞
硬件清单
「一台小主机」
：我的主力，安静又省电。
「NAS系统（宿主机）」
：群晖、飞牛、PVE都行，我个人偏爱PVE的灵活性，这次就用它演示。
「虚拟系统」
：我选择了Ubuntu 25作为运行环境，界面越来越美了。
「良好的网络环境」
：你懂的，这是顺利安装的隐形基石。
模型选择
需要一个订阅的语言大模型。当然，用免费的也行，但分分钟可能“烧钱”哦！ 我选用的是
「MiniMAX模型」
，也是OpenClaw官方推荐的国内模型。记得开通
coding plan
，用量大还实惠。
通讯通道
我选择
「Discord」
作为交互通道，这样就能和家人一起在聊天中控制OpenClaw了，特别方便。

安装步骤大致如下，我们一步步来。
在PVE中安装Ubuntu
安装过程
安装虚拟机不是本次的重点，相信玩PVE的朋友都没问题。
❝
其实只要是虚拟机都可以，无论你是用群晖、飞牛还是其他，原理都一样。

设置系统权限（免密操作）
OpenClaw需要较高的系统权限，为了避免反复输入
sudo
密码，我们先开启免密操作。
编辑 sudoers 配置
sudo visudo
添加免密权限
在文件末尾添加以下内容（记得把
leolee
替换成你自己的用户名哦）：
leolee ALL=(ALL) NOPASSWD: ALL
保存并退出
如果你用的是
nano
编辑器：按
Ctrl+O
保存，
Enter
确认，再按
Ctrl+X
退出。
如果是
vim
编辑器：输入
:wq
然后回车。
（可选）安装SSH
这步主要是为了访问方便。虚拟机的图形界面有时会卡顿，用SSH工具管理就流畅多了。
安装基础工具与 OpenSSH
更新系统并安装远程管理需要的组件。
#
更新系统包列表
sudo apt update
#
安装基础下载工具
sudo apt install -y curl
#
安装并启动 OpenSSH 服务（方便用电脑远程管理）
sudo apt install -y openssh-server
验证 SSH 服务状态
#
检查 SSH 服务是否在运行
sudo systemctl status ssh
#
如果没启动，用这个命令启动它
sudo systemctl start ssh

这步不是必须的，但为了安装过程更顺畅，建议有一个良好的网络环境。一个小提示：如果路由器能设置全局代理，会让网络环境稳定不少。 我这里把虚拟机的网络网关指向了路由器。

安装OpenClaw主体
终于进入正题了！网上有些教程会先配置Claude Code，再用它来装OpenClaw。但考虑到Claude Code的成本和可用性，我决定手动“硬刚”OpenClaw的配置。😂
下面我都会在SSH工具里操作。
跟着官网教程走
一手资料最可靠！打开OpenClaw官网：https://openclaw.ai/#install有了SSH工具，直接复制粘贴命令就行。
安装命令：
curl -fsSL https://openclaw.ai/install.sh | bash
因为项目更新快，如果一个方法报错，可以试试另一种，比如：
npm i -g openclaw && openclaw onboard

（可选）启动WebUI
这步主要是为了能在局域网的其他电脑上通过浏览器管理OpenClaw。
启动网关：
openclaw gateway --bind lan --port 18789
在本地电脑上做SSH端口转发（假设Ubuntu虚拟机IP是192.168.1.37）：
ssh -N -L 18789:127.0.0.1:18789 leolee@192.168.1.37
然后启动仪表盘：
openclaw dashboard
打开终端返回的地址+令牌，就能在浏览器里访问WebUI了。
配置通讯通道 (Channel)
正如开头所说，我常用Discord，所以以它为例。其他通道的教程网上已经有很多啦。
在终端运行：
openclaw config
找到Channel配置，选择Discord，终端会显示详细的指引：
Discord开发者后台配置
打开Discord开发者门户：https://discord.com/developers/applications
创建完应用后，点击左侧
Bot
，然后点击
Token
旁边的
Reset Token
。
拿到Token后，粘贴回OpenClaw的终端即可。

复制生成的链接，在浏览器中打开，将Bot邀请到你刚创建的服务器。

配置正确的模型
经过一番调试，Bot终于在Discord回话了，但……是错误信息！
排查发现是模型配置有问题。在终端重新输入：
openclaw configure
按以下流程选择：
Model
MiniMax
MiniMax M2.1
再次输入你的API Key。
然后在模型列表中，找到
minimax-cn/MiniMax-M2. 1
，
「用空格键选中」
。
按回车，选择
continue
，完成设置。
如果还不行，那很可能是
baseurl
不对。继续在终端输入：
nano ~/.openclaw/openclaw.json
找到MiniMax对应的
baseurl
。我发现这里错了：
https://api.minimax.io/anthropic
，需要改为：
https://api.minimaxi.com/
保存并退出nano编辑器。
「重启网关让配置生效」
：
openclaw gateway stop
openclaw gateway --
bind
lan --port 18789
