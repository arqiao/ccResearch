01 我想要的是“交付”，不是“陪聊”
AI 现在最容易让人误判的点是：它回答得很像，但事情没推进。
我对工具的标准很直白：
能不能替我把任务往前推一段
能不能留下记录，方便复盘
能不能我不在电脑前也能继续推进
一句话：
我需要的是一个“可控的助理”，不是一个“高情商搜索框”。
02 命令名别被带跑偏：装的时候是 molt.bot，跑起来是 clawdbot

03 点火过程（我把“翻车的地方”都写上）
1、先装 WSL2，PowerShell 里一句话：wsl --install
别纠结环境，先把“能跑起来”当第一目标。
2、装好之后用wsl命令进入Ubuntu，装基础工具
拷贝命令：
sudo apt update
sudo apt install -y curl git build-essential ca-certificates
少装软件，多装底座。底座稳了，后面全是顺滑。
3、用 nvm 管 Node（这一步决定你后面会不会踩坑）
拷贝命令：
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
Node 版本就是门槛，别硬扛。
4、直接上 Node 24（关键）
nvm install 24
nvm use 24
node -v
npm -v
别在 22 上“侥幸能用”，一次升级，省三次排障（血泪教训）。
5、修 PATH（避免“装好了但命令找不到”）
拷贝命令：
echo'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
很多“玄学问题”，其实都叫 PATH。
6、用官方脚本安装clawdbot
curl -fsSL https://molt.bot/install.sh | bash -s -- --verbose
能用脚本就别手搓，手搓的快乐只属于没时间的人。
看到 onboarding 界面，就算点火成功一半了。

04 让 WSL2 真正“能上网”：代理的正确打开方式
WSL2 默认 NAT，一个核心原则：
WSL里别用localhost 指Windows 代理，要用Host网关 IP。
代理配置直接用这段：
HOST_IP=$(ip route | awk '/default/ {print $3}')
export http_proxy="http://$HOST_IP:10809"
export https_proxy="http://$HOST_IP:10809"
export ALL_PROXY="socks5h://$HOST_IP:10808"
然后用curl -I https://www.huajutech.com.cn --max-time 10测试下网络是否可用
ping 能通不算通，curl 能通才算“真上网”。
05 进阶：接入飞书后，我能“远程遥控”本地 Bot 📲
这一步很关键。
你可以这样理解：
WSL2 里的 Clawdbot：发动机（本地常驻执行）
飞书：遥控器（消息就是指令）
你在手机发一句，它在本地跑完把结果回传
它把“电脑里的工具”变成“随身的助手”。飞书开放平台需要建应用，把权限和事件回调配置好，至于Clawdbot端，交给他自己配置就好，参考
https://github.com/m1heng/clawdbot-feishu
。
把入口放到飞书/钉钉/QQ，随时可以遥控指挥助理做事情。
06 到这里才开始好玩：它到底能干什么？
片段 1：查假期，顺便把“怎么请最划算”给你算出来
好用的助理会回答下一问，而不是只回答这一问。
片段 2：整理文件，从杂物堆变成项目库
它做事方式很像靠谱同事：
先给计划，再执行，再汇报。更牛的是它会自动写个脚本来完成这个事情，而不是告诉用户一个脚本，然后用户再安装运行环境把脚本跑起来，给结果而不是方法，这才是助理该干的事。
先给 Plan 再动手，这是人和机器都能安心合作的方式。
07 我最喜欢的一段：新闻图还能“自我进化” 🖼️🧬
这才是我觉得它像“Agent”的地方。
不是一次性生成，而是能跟你一起迭代。
第一步：它先当编辑，再当设计
它先找新闻、列来源、做摘要，然后出第一版封面图 V1。
先把信息做对，再谈好不好看。
第二步：你别返工，你只提“改稿意见”
比如：
信息密度降一点
标题层级更清楚
留白干净，像公众号封面
风格统一，做成可复用模板
然后它继续出 V2/V3。
评价标准比灵感更值钱。标准一旦明确，图会自己变好。
安装完插件后生成的第一版新闻：居然是乱码
连着像几张他自己进化后生成的新闻图：
第三步：你会发现它开始“沿着你的审美收敛”
AI 生成图最怕的不是丑，是不可控。能迭代到可控才算进入生产。
09 三条小原则：让它更像“能用的助理”
1）先计划再执行
计划不是啰嗦，是减少返工。
2）创意任务默认三轮
第一版求方向，第二版求结构，第三版求质感。
3）权限最小化
能做很多不代表应该做很多。
结尾：分水岭不是“装没装”，是“你每天会不会用”
Agent 最容易变成“装完就吃灰”的东西。
解决办法只有一个：给它一个每天会发生的任务。
每天一份“新闻摘要 + 封面图（可迭代）”
每周自动整理下载目录
飞书随口一句话，把任务推进到可交付
最后送你一句我现在践行的标准：
最好的 AI，不是更会说话，是更会把事情办完。
