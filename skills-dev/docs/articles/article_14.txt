 id="js_content" style="visibility: hidden; opacity: 0; "> 今天早上刷 GitHub 的时候，看到一条消息让我眼前一亮： Obsidian 的 CEO Steph Ango（GitHub 用户名：kepano）竟然亲自给 Claude Code 写了三个 Skill。 我第一反应是： 这个 CEO 也太硬核了吧？ 作为一个 Obsidian 的重度用户，我当然不能错过这个新功能。下载、安装、测试，整个过程不到 10 分钟。 然后我做了一件事： 让 AI 给我画了张甄嬛传的人物关系图。 14 个主要角色，15 条关系线，谁跟谁是一伙的、谁跟谁有仇，用颜色一标，清清楚楚。 关键是，我一行代码都没写，就跟 Claude 说了句话： "帮我用 canvas 画一张甄嬛传的人物关系图到 xx 文件夹" 3 分钟后，这张图就出现在我的 Obsidian 里了。 这就是今天要分享的内容： Obsidian CEO 创建的三个 Claude Code Skill，以及如何用它们快速制作知识图谱。 一、为什么 Obsidian CEO 要亲自下场？ Steph Ango 是谁？ 很多人可能不知道，Steph Ango（网名 kepano）成为 Obsidian CEO 的过程挺传奇。 他最开始就是个超级粉丝，没事就在社区里提建议、吐槽哪里不好用。 后来 Obsidian 两位创始人 Shida Li 和 Erica Xu 看这人挺靠谱，直接邀请他进团队帮忙改主题。 结果一不小心，做着做着就成 CEO 了。 从粉丝到 CEO，这个路径本身就很 Obsidian： 开放、社区驱动、注重用户反馈。 为什么要创建这三个 Skill？ Claude Code 的 Skill 机制是这样的： 你可以通过一个 SKILL.md 文件，告诉 AI 如何处理特定类型的任务。 比如： 如何创建 Obsidian 风格的 Markdown 文件 如何编辑 .base 文件（类似数据库视图） 如何生成 .canvas 文件（无限画布） kepano 创建的三个 Skill，正好覆盖了 Obsidian 最核心的三种文件格式： obsidian-markdown ：处理 Obsidian 风格的 Markdown（wikilinks、callouts、properties 等） obsidian-bases ：创建和编辑 .base 文件（表格视图、过滤器、公式） json-canvas ：生成 .canvas 文件（思维导图、流程图、关系图） 这三个 Skill 一旦装上，Claude Code 就能"听懂"Obsidian 的语言了。 你不需要告诉它"用双方括号创建链接"，它自己就知道。 你也不需要解释"canvas 文件的 JSON 结构"，它直接就能生成。 这就是 AI 时代的"工具适配"。 AI + 知识管理的新趋势 过去我们用 Obsidian，主要是手工整理笔记、画图、建立知识网络。 现在有了 AI，我们可以： 说一句话，让 AI 生成思维导图 给一段文字，让 AI 提取关系网络 提供一些数据，让 AI 自动生成数据库视图 这不是替代人工整理，而是 把人从重复劳动中解放出来，专注于思考本身。 二、10 分钟实战教程：从零开始画甄嬛传关系图 好了，理论讲完了，现在进入实操。 跟着我的步骤来， 10 分钟你也能画出自己的关系图。 Step 1：准备工作（5 分钟） 1.1 安装 Claude Code 如果你还没装 Claude Code，先去官网下一个： 🔗 https://code.claude.com 装完打开终端，敲一下： claude 能看到 Claude 跟你打招呼，就 OK 了。 1.2 安装 kepano 的三个 Skill 这步稍微有点技术含量，但我保证很简单。 第一步：下载 Skill 文件 打开 kepano 的 GitHub： 🔗 https://github.com/kepano 找到这三个仓库（直接搜 obsidian-markdown、obsidian-bases、json-canvas），把 SKILL.md 文件下载下来。 第二步：复制到指定位置 在你电脑上找到（或创建）这个文件夹： ~/.claude/skills/ 把刚才下载的三个文件夹（每个里面有个 SKILL.md）都扔进去。 最终结构是这样的： ~/.claude/skills/ ├── obsidian-markdown/ │ └── SKILL.md ├── obsidian-bases/ │ └── SKILL.md └── json-canvas/ └── SKILL.md 第三步：验证安装 回到 Claude Code，随便问一句： 你有 json-canvas 这个 skill 吗？ 如果 Claude 回答"有"，就说明装好了。 或者你直接告诉 Claudecode Step 2：在 Obsidian 里准备文件夹（30 秒） 打开 Obsidian，随便找个地方新建一个文件夹，比如叫 AI-Canvas 。 然后右键这个文件夹，选"复制文件路径"。 比如我的路径是： /Users/crazymax/Documents/Obsidian/AI-Canvas 复制好，待会儿要用。 Step 3：跟 Claude 说你想要什么（30 秒） 回到 Claude Code，直接说： 帮我用 canvas 画一张甄嬛传的人物关系图，保存到 /Users/crazymax/Documents/Obsidian/AI-Canvas 记得把路径换成你自己刚复制的那个。 然后就等着，Claude 会开始干活。 Step 4：查看和调整（2 分钟） 一两分钟后，Claude 会说文件生成好了。 回 Obsidian，打开刚才那个文件夹，会看到一个新文件： 甄嬛传人物关系图.canvas 双击，完整的关系图就出来了。 找不到“all in one/06_writingflow/推特/images/甄嬛传人物关系图-详细截图.png”。 我这张图里有： 14 个角色 ：甄嬛、皇帝、皇后、华妃、沈眉庄、安陵容、温实初、果郡王…… 15 条关系 ：主仆、师徒、情敌、政治联盟、暗恋…… 不同颜色 ：每个阵营一个颜色 标签说明 ：每条线上都写清楚是什么关系 觉得哪里不对，直接在 Canvas 里改： 拖节点换位置 双击改内容 右键加新连线 AI 负责快速搭框架，你负责精修细节。这才是正确的协作方式。 三、Claude 是怎么设计这张图的？ 拿到图之后，我仔细看了下 Claude 的思路，发现还挺讲究。 人物分层 Claude 把 14 个角色分成了三圈： 核心圈（5 人） ：甄嬛、皇帝、皇后、华妃、果郡王 主线人物，放中心 节点最大 第二圈（6 人） ：沈眉庄、安陵容、温实初、端妃、曹贵人、敬妃 重要配角 围着核心圈转 节点中等 外圈（3 人） ：苏培盛、剪秋、浣碧 关键时刻出手的角色 节点最小 颜色策略 不同阵营用不同颜色： 红色：皇帝阵营 紫色：甄嬛阵营 蓝色：皇后阵营 橙色：华妃阵营 绿色：中立/墙头草 一眼就能看出谁跟谁是一伙的。 关系线设计 连线也不是乱画的： 主仆关系 ：实线 + 箭头（指向主人） 情感关系 ：虚线 + 双向箭头 政治联盟 ：粗实线 暗斗 ：加文字标签说明 比如甄嬛和果郡王之间那条线，标着"爱情"两个字，一看就懂。 这种可视化，比你读 100 页剧情介绍都清楚。 四、除了宫斗剧，还能画什么？ json-canvas 这个 Skill 能干的事多了。 几个场景： 1. 项目规划 跟 Claude 说： 画个产品开发流程图，需求分析、设计、开发、测试、上线 它会给你生成标准流程图，每个阶段写清楚要干啥、谁负责。 2. 技术架构 说： 画张前后端分离架构图，前端、后端、数据库、缓存都要 Claude 按标准分层给你画出来。 3. 读书笔记 说： 《人类简史》核心概念关系图 Claude 会把"认知革命"、"农业革命"、"科学革命"这些概念提取出来，把它们之间的关系连上。 4. 整理想法 说： 整理下我最近的创业想法，AI 工具、知识管理、自动化三个方向 零散的想法，Claude 帮你组织成结构化思维导图。 核心逻辑就四步： 你说要画什么 → Claude 提取关键元素 → 自动生成图表 → 你在 Obsidian 里打开调整 五、门槛降到零了 以前画这种关系图，要么用 Xmind、ProcessOn 这些专业软件，要么在 Obsidian 里一个个拖节点。 都挺费劲的： 要学软件怎么用 布局要调半天 细节改来改去 现在有了 AI + Skill： 你不用懂 JSON 格式。 不用记 Obsidian 语法。 只要说清楚"我想要什么"，AI 就能给你做出来。 过去只有专业人士才能搞定的事，现在普通人也能干。 而且效果还不差。 写在最后 kepano 这三个 Skill，只是个开始。 但它让我看到了一件事： AI 不是来抢工具饭碗的，是来让工具更好用的。 好的 CEO 不是高高在上发号施令，而是亲自下场解决用户的真问题。 知识管理的未来，不是功能越来越复杂，而是体验越来越简单。 如果你在用 Obsidian，建议试试这三个 Skill。 如果你还没用过 Claude Code，现在是个好机会。 最后，如果你也画了关系图（甄嬛传或者别的），欢迎留言晒图。 我很好奇 AI 还能帮我们画出什么有意思的东西。 如果这篇文章对你有帮助，欢迎扫码链接。我会持续分享 AI 工具的实战经验。
