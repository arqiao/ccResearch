标题: 一切皆可Agent Skills，无处不在的AI Agent会替代业务流程吗？

 id="js_content" style="visibility: hidden; opacity: 0; "> 一篇文章，看懂业务流程、工作流、Agentic Workflow、AI Agent、Agent Skills之间的区别与联系。没有谁替代谁，只有融合发展。 全文约4900字 阅读时间8分钟 你公司的ERP审批流程，点十几个按钮，转三个部门，等五天才能报销一张发票。 隔壁创业公司，一句话跟AI说帮我处理这笔报销，三分钟搞定。 老板叫你进办公室：咱们那套三百万的工作流系统，是不是可以扔了？ 先别慌。我告诉你，这事儿没那么简单。 翻车现场 2023年3月，AutoGPT在GitHub上火得一塌糊涂。一个AI自己拆任务、自己写代码、自己调试，科幻片照进现实。技术圈集体高潮，标题党满天飞："程序员要失业了""自动化的终极形态来了""工作流已死"。 2年多过去了，这些Agent在生产环境翻车翻得有多惨？用过的人都懂。 幻觉乱飞、死循环跑不出来、不知道自己在干啥、把测试数据写进生产库…… 早期AutoGPT那一代野生Agent，就是个定时炸弹。我见过最离谱的案例，是一个Agent被要求优化服务器性能，结果它把客户数据库给删了。理由是数据库太大影响性能。 但故事到这儿没结束。 吴恩达在2024年初说了句话：别天天想着更强的模型，设计好的工作流更重要。他团队做了个实验，给GPT-3.5套上一个反思-规划-执行的框架，在代码生成任务上直接干翻了裸跑的GPT-4。 看到没？流程结构比模型大小更要命。 真正的问题从来不是Agent能不能取代工作流，而是谁来给Agent设计工作流。 先搞清楚几个概念 这场讨论最要命的地方，就是面对一堆术语，每个人理解得都不一样。 业务流程就是你公司里那些"必须这么干"的事儿。员工入职走哪些手续、合同审批过哪些关卡、发票报销填什么单子。这东西是企业的骨架，是合规的底线，是前人用血泪踩坑踩出来的经验。 工作流，就是把业务流程塞进电脑里。纸质表单变成OA系统的流转节点，人工审批变成系统自动检查。Camunda、Activiti这些工作流引擎，就是把企业规则写成代码，让机器照着办。死板，但稳。 AI Agent是个会琢磨的执行者。你跟它说个目标，它自己想办法完成。要查数据库就调API，要发邮件就调邮件接口，遇到问题还能停下来想想我是不是搞错了。最妙的是，你不用提前把每一步都写死。 Agent Skills就是Agent的工具箱。"查客户信息"是一个Skill，"生成PDF报告"是一个Skill，"发钉钉消息"也是。每个Skill都是原子能力，Agent按需组合。 Agentic Workflow才是今天的重点。说白了，它是个指挥系统，把Agent、大模型、各种工具、甚至传统工作流全接进来，根据实际情况动态编排。谁该干什么、什么时候干、出问题了怎么兜底，都归它管。 这五者之间的关系，不是谁替代谁，而是层层嵌套。Agent Skills是零件，Agent是工人，Agentic Workflow是项目经理，业务流程是建筑图纸。 业务流程会消失吗？ 我先说个残酷的事实：业务流程这玩意儿，跟人类社会一样古老。 苏美尔人刻泥板记账？业务流程。福特的流水线？业务流程。你早上起床刷牙洗脸吃早饭？还是业务流程。只要干活有先后顺序，业务流程就死不了。 有人可能会问：AI Agent这么聪明，还要啥流程啊？让它自由发挥不就完了？ 行，你试试把银行转账审批交给一个完全自主的Agent，看监管部门怎么收拾你。你让Agent自己决定要不要给客户退款，法务部能把你喝茶喝到吐。 业务流程的本质，从来不是为了限制效率，是为了管风险。 公司的合同审批为什么非要走三级？不是为了恶心人，是因为十年前有个傻子签了个坑爹合同，公司赔了五百万。财务报销为什么死抓着发票不放？因为税务局来查账能要你命。这些规则，都是真金白银换来的。 Agent再怎么聪明，本质上就是个概率模型。常规场景它能处理80%，但剩下20%的边缘情况呢？客户突然变卦、供应商跑路、系统宕机、数据对不上——这些训练数据里没见过的破事儿一来，Agent立马懵逼。 所以问题不是要不要业务流程，而是业务流程怎么活下去。 以前写在文档里，印成手册，新人培训三个月才记得住。后来固化成工作流系统，节点写死了，想点错都点不了。现在要进化成Agent能理解的规则。 打个比方。传统工作流是铁轨，火车只能顺着轨道跑，想变道？没门。Agentic Workflow是道路系统，红绿灯、限速牌、单行线该有还得有，但车能根据路况选路线。规则在，但活了。 Agent是工作流的变种 我必须纠正一个误区：Agent不是什么新物种，它就是工作流的另一种存在形式。 以前一个报销流程要拆成十几个节点：提交申请→部门审批→财务审核→出纳付款。每个节点是个独立的自动化单元，中间靠系统硬编码串起来。 现在一个Agent可以把这十几个节点的活儿全包了。不光能执行每个步骤，还能根据情况调整顺序。发票不到一千块？跳过部门领导，直接财务批。发票有问题？暂停流程，自动发消息让你补材料。供应商在黑名单上？触发风控，通知法务。 一个主动式Agent，约等于一个自适应的微型工作流引擎。不再是流程里的一颗螺丝钉，而是能装下多个工作流逻辑的容器。更绝的是，多个Agent之间还能协作。 LangChain的CEO说得很明白：完全自主的Agent信不过，得给它加护栏。啥护栏？就是把业务规则、审批逻辑、异常处理，用Agent听得懂的话告诉它。 实际例子。你是电商公司的客服主管，要处理退款。传统工作流长这样： 客户申请退款 → 检查订单状态 → 检查退款原因 → 判断金额 → 自动/人工审批 → 退款到账 → 发通知 每个箭头都是硬编码。遇到特殊情况？系统直接认怂，转人工。 换成Agent： 客户："我要退款，商品有质量问题" Agent: - 查订单，拿信息 - 查物流，确认收货了 - 分析原因（质量问题），判断得人管 - 自动生成工单，标优先级 - 通知人工客服，退款单提前填好 - 2小时没人理？升级给主管 - 退款完事后，记进客户档案，打上"质量敏感"标签 差别在哪？Agent不是干掉了工作流，而是把工作流变成了能组合、会判断、可自适应的单元。 Agent Skills：把能力掰成零件 很多人搞不懂Agent Skills。其实很简单，就是把API包装成Agent听得懂的话。 以前调个数据库查询接口，得写代码： python db.query("SELECT * FROM customers WHERE id = ?", customer_id) 现在你把这玩意儿包装成Skill： Skill名称: 查客户信息 干嘛用的: 根据客户ID查基本信息、历史订单、信用分 需要啥: customer_id（一串字符） 给你啥: 客户对象（JSON） Agent看到这描述，就懂了：哦，我要了解客户的时候，就调这个。关键是，Agent能自己判断啥时候该用。你不用写如果用户问客户信息就调这个API，Agent自己通过语义理解匹配。 Agent Skills的价值就是：标准化 + 乐高化。 过去每个系统、每个部门都有自己的一套接口。业务人员想搞个跨部门的自动化？得找三个技术团队开会，排期一个月，联调两周。运气好的话，三个月后能上线。 现在把各部门的核心能力都封装成Agent Skills，扔进企业技能库。要啥能力，Agent自己挑。销售要查库存？调库存Skill。财务要对账？调财务Skill。客服要查物流？调物流Skill。 这已经不是技术问题了，这是 企业能力的重新排列组合 。 想象你公司有个技能超市，里面躺着两百个标准化Skills。从"查客户信息"到"生成财报"，从"发审批通知"到"调大模型写文案"，应有尽有。每次来新需求，不是从零开发系统，而是用Agent把需要的Skills串起来。 微软说用AutoGen开发，代码量能减少4倍。但真正牛逼的不是少写代码，而是业务人员也能参与。 你不需要懂Python，只要跟Agent说清楚：我要实现这个目标，把每步需要啥能力描述明白。Agent自己去技能库里找对应的Skills，自己测试能不能跑通。技术团队的角色从实现者变成能力提供者。 简单地讲，Agent Skills就是把公司里那些只存在于老员工脑子里的隐性知识，掰碎了，标准化，放进库里。 Agentic Workflow：终极大管家 现在到重头戏了：Agentic Workflow到底是个啥？ 很多人以为就是多个Agent一起干活。错了。它是个全新的编排范式，目标是把所有执行单元班阔Agent、大模型、传统工具、甚至老工作流等全部调度起来。 场景：你是制造企业的供应链总监，接到个紧急订单。这单子要协调七个部门、三个外部供应商、两个物流商。传统做法？开会、发邮件、打电话、Excel满天飞，三天能搞定就烧高香了。 用Agentic Workflow： 订单进来 → - Agent A（需求分析）: 解析订单，拆成BOM清单 - Agent B（库存检查）: 并行查各仓库库存，算缺口 - Agent C（供应商协调）: 给三个供应商发询价单，2小时不回算你输 - *传统工作流*：走采购审批流程（这块不能乱） - Agent D（物流规划）: 根据供应商位置优化配送 - Agent E（生产排期）: 调ERP，插入生产计划 - Agent F（风险监控）: 盯着各环节，发现延迟立马吼 - *人工决策点*：成本超预算15%，暂停，找主管批 - Agent G（客户沟通）: 自动生成进度报告，每天发客户 这流程里，有Agent自己做决定，有传统工作流的死规矩，有外部API调用，有人工拍板的节点。Agentic Workflow是总指挥，不管每个单元内部咋玩的，只管把它们高效串起来。 传统工作流引擎干不了这个。它只能处理确定性路径：A干完去B，B干完去C。遇到"根据供应商反馈动态调整"这种需求？对不起，改需求、改代码、测试、上线，两周起。 Agentic Workflow的杀手锏是动态编排。不提前把路全写死，而是定好目标、约束、可用资源，让系统自己找最优解。供应商A不回消息？立马切备选。物流费超了？要么重算路线，要么延迟发货。 但这灵活性是有代价的：控制力下降。所以必须设计护栏。 LangGraph的做法是把流程画成图（Graph）。节点可以是Agent、可以是函数、可以是人工审批点。边定义流转规则，但这规则能是动态的。系统跑起来之后，根据每个节点的输出决定下一步。 微软AutoGen的路子是多智能体编排。每个Agent有明确的角色，通过消息传递协作。一个Agent搞不定了，可以@另一个Agent求助。这模式很像人类团队干活，但速度快几个数量级。 Agentic Workflow的终极价值，是让企业复杂的业务流程具备了"软件定义"的能力。 以前业务变了就得改系统，几个月项目周期是常态。现在业务规则调整，可能只要重新配置Agentic Workflow的编排逻辑，一天搞定。这种敏捷性，在供应链、金融交易、客户服务这些快速变化的领域，值老钱了。 到底谁会被干掉？ 回到开头的问题：Agent会替代业务流程吗？Agent Skills会替代工作流吗？ 我的答案：都不会，但都得进化。 业务流程死不了。只要公司在，只要干活有顺序，业务流程就得在。但它的形态会变：从纸质文档到工作流系统，再到Agent能理解的规则。流程变灵活了，但底层逻辑"事情该咋干"永远在。 工作流不会被Agent Skills替代，因为它们根本不在一个层面。Agent Skills是能力单元，工作流是执行路径。以后的工作流会由Agent来跑，而不是硬编码的系统节点。一个Agent能装下多个工作流的逻辑，但工作流本身的价值“把业务逻辑结构化、可复用”跑不掉。 真正要挂的是那些只会机械执行、不会灵活变通的自动化系统。 比如简单重复、规则固定的RPA任务，Agent分分钟干得更溜。那些需要人判断但其实就是套模板的审批环节？Agent直接接手。那些跨系统协调但只会傻等的集成中间件，Agentic Workflow让它们显得又笨又慢。 所以，被淘汰的不是"流程"，是"僵化"。 我们该咋办？ 如果你是技术负责人，现在最该干的不是推翻现有系统，而是盘点企业能力，建Agent Skills库。 把各系统的核心能力抽出来，定义成标准化Skills。财务系统有啥能调的？ERP有啥能查的？CRM有啥能触发的？先把武器库建起来。 然后挑几个高频、复杂、但风险可控的场景，试点Agentic Workflow。别一上来就搞供应链大改造，先从客服、内部协同这种容错率高的场景练手。 最重要的是培养流程工程能力。以后的关键岗位不是会写代码的程序员，而是会设计Agent协作模式的流程架构师。这人得懂业务、懂AI能力边界、也得懂怎么设计护栏和兜底。 如果你是业务负责人， 别指望Agent一夜之间解决所有问题，但也别死守旧系统。主动参与Agent Skills的定义。你最懂业务逻辑，你最清楚哪些环节能自动化、哪些必须人工把关。这些知识，是建企业级Agentic Workflow的核心资产。 终局是怎样的？ Gartner预测2028年15%的工作决策由AI Agent自主完成。这不是画饼，已经在发生了。 但这未来不是人类靠边站，Agent全自动，而是人类定目标定约束，Agent负责执行和优化。 看个电商公司的一天： 早上，市场部Agent发现竞品搞促销，自动分析了价格策略和库存，生成三套方案，发给运营总监。总监选方案二，批准。 中午，供应链Agent发现畅销品库存告急，自动联系供应商、调物流排期、更新网站预计到货时间。全程无人工，因为在"授权范围"内。 下午，客服Agent处理三百个售后，二十个退款自动搞定，五个复杂纠纷转人工，背景信息和建议方案已经准备好了。 晚上，财务Agent生成当日经营数据，标出三个异常指标，追溯可能原因。CFO看完，决定明天早会讨论。 这就是Agentic Workflow或者说Agentic AI（人机协作）的终局：业务流程还在，但执行层变成了Agent海洋。人类角色从操作工变成指挥官和决策者。 业务流程不会消失，会以更高级的形态存在：不再是死规矩，而是活策略；不再是固定节点，而是灵活编排；不再是系统的枷锁，而是智能的框架。 这场变革的本质不是AI替代人，而是让人从流程的奴隶变成流程的主人。 那些让你抓框的审批流、让你崩溃的跨部门扯皮、让你熬夜的数据整理，它们不会消失，但会被Agent接手。你能把精力放在真正需要创造力、判断力、同理心的地方。 说到底，这不是Agent的胜利，是人的解放。 现在就断言"Agent会完全取代XX"，跟当年说"互联网会干掉实体店"是一样的，没有绝对的取代，只有更好地融合。技术演进从来不是简单的你死我活，而是复杂的元素融合。 业务流程、工作流、Agent、Agentic Workflow，它们会共存很久，会在碰撞中融合，会在实践中进化。重要的不是站队"挺流程"还是"挺Agent"，而是搞清楚它们各自的价值，找到最适合你的组合。 工具、技术、方案是死的，组织、团队和人是活的。聪明人用Agent优化流程，看不懂的人用Agent制造混乱。 看懂终局，你就会有更好的选择。 全文完 王吉伟频道新书《一本书读懂AI
