 id="js_content" style="visibility: hidden; opacity: 0; "> 最近 AI 圈有个词特别火—— Skill 。 不是英语课本里的“技能”，而是 Claude Code 里的一个新概念： 你把自己的工作流写成一份标准文档（SKILL.md), AI 就能像老员工一样，严格按照你的 SOP 一步步执行。 不用写代码。不用学编程。你只需要把“你平时怎么干活”讲清楚，AI 就能帮你干。 听起来像画饼？ 上周刚结束的 全球首场 Claude Agent Skills 黑客松 ，用 38 个真实作品证明了——这不是画饼，这是正在发生的事。 宝妈做了法律合同审查 Skill，十年没碰代码的新疆农民做了番茄视觉质检 Skill，建筑工程师做了新闻海报生成 Skill…… 两天直播，38 个 Skills 作品，绝大多数开发者是零基础的“普通人”。而我们 SuperAI 编程 团队，也带着自己的 Skills 杀了进去。 今天这篇文章，聊聊这场黑客松里我们看到了什么，以及——我们自己做了什么。 为什么说 2026 是“Skill 元年”？ 这次黑客松由 AI 产品黄叔 主理，核心主题就是 Claude Agent Skills 。 先快速科普一下 Skill 到底是什么—— 传统的 AI 对话是“一问一答”，你问一句它答一句，聊完就没了。而 Skill 是一套可复用的工作流程 。你把行业经验、操作步骤、质量标准写进一个 SKILL.md 文件，Claude 就会像一个训练有素的员工一样，严格按流程执行，还能调用各种 API、生成文件、操作数据。 一次编写，无限复用。 这才是 Skill 真正的杀手锏。 你写了一个“生成环评报告”的 Skill，以后每次只要输入 /环评 ， AI 就自动跑完全流程。不用重复教，不用重复说，它记住了你的 SOP。 这 38 个作品，让我看到了三个非常明显的趋势： 趋势一：从“通用助手”到“领域专家”。 大家不再满足于让 AI 当个聊天机器人，而是把自己十几年的行业经验编码成 Skill，让 AI 变成自己的“数字分身”。环评人写环评 Skill，律师写合同审查 Skill，药企员工写控价巡查 Skill—— 每个人都在用 Skill 封装自己最值钱的经验。 趋势二：从“玩具”到“生产力工具”。 这些 Skill 不是 Demo，是真的在用的。药企控价 Skill 替代了每天 30 分钟的人工巡查，天然气日报 Skill 已经被公司内部采纳，面试助手 Skill 现场被猎头盯上…… Skill 正在成为新的生产力单元。 趋势三：从“单兵”到“智能体团队”。 黄叔在第二天演示了 Claude Code Agent Team——多个 Skill 协作，互相博弈、打分、修改，还能把成功经验沉淀到知识库。 Skill 不只是工具，它在进化。 SuperAI 编程的 Skills 作品 我们团队这次有三位成员带着作品参赛：十七°、Ruby 和 Li Smith。当然，我们团队远不止这三个人，但这次冲在前线的是他们仨。 每个人的 Skill，都代表了一种不同的“Skills 思维”。 十七°: Video Factory——当 Skill 学会“生 Skill” （没错，就是我） 别人做的是一个 Skill， 我做的是一个 Skill 工厂。 你给我一个主题，甚至只需要一句话，系统就能自动完成： 剧本生成 → 角色设计 → 分镜绘制 → 运镜合成 → 配音配乐 → 字幕成片 最终输出一部完整的“解说漫”视频。全程零人工干预。 听起来很简单对吧？但这背后的工程量，用黑客松主持人的原话说——“工程量浩大”。 这个系统的核心设计理念是：Skill 的嵌套与编排。 我没有写一个巨大的 Skill 来完成所有事情，而是设计了一套 三层 Skill 架构 ： 顶层：总指挥（Video Factory） —— 就像一个导演，负责协调下面三个“工作室” 第一层：故事工坊 —— 负责“编剧”工作 你只需要给一个主题（比如“一个程序员的日常”）, AI 就会自动写出完整的故事：先列大纲，再设计角色（长什么样、什么性格），然后写剧本，最后拆成一个个镜头。就像电影拍摄前的剧本准备工作，全自动完成。 第二层：视觉工坊 —— 负责“画画”工作 根据剧本把画面画出来。系统会先给每个角色设计形象和表情包（9 种表情，保证角色在所有画面里长得一样），然后按照分镜表逐个画出场景。还有一个“质检员”会检查每张画——构图对不对、角色像不像、情绪到不到位，不合格就重画。 第三层：视频工坊 —— 负责“做视频”工作 这是 解说漫的核心部分 。把静态的画面变成动态视频，主要靠 FFmpeg 专业运镜技术 （推、拉、摇、移、Ken Burns 等电影级镜头运动），让静态图“动”起来。同时配上智能配音（16 种声音自动匹配角色）、背景音乐、音效，调整画面色调和音量，最后加上字幕——一键合成完整视频。 另外，系统还接入了 AI 视频生成功能，可以生成真正的动态视频；也支持 3Blue1Brown 那种数学动画风格（用 Manim 引擎渲染）。 目前已经能做出两种视频： 「解说漫」 ——这是主力产品。核心技术是 FFmpeg 专业运镜 （就像摄影师拿着相机对着画面推拉摇移），让静态画面产生电影感的动态效果，再配上 AI 配音和字幕。很多短视频平台上的漫画解说就是这种风格。现在从输入一句话到输出成片，已经完全自动化了。 另外，系统还接入了 AI 视频生成功能 ，可以生成真正的动态视频片段。 「数学动画」 ——3Blue1Brown 那种风格的数学可视化动画，用专门的 Manim 引擎渲染。已经成功做出了概率论动画。 说实话，这个系统离我心中的“终极形态”还有距离。下一步要做的 Remotion 动态渲染引擎（可以做出更炫酷的动效）和素材混剪模式（像真人 UP 主那样剪视频），架构设计和技术文档已经写好了，就差动手做了。 不过，架构已经搭好了。 一个总指挥 + 三个工作室 + 十几个小工具，每个小工具都可以单独用，组合起来就是一条完整的流水线。想单独设计角色？用角色设计工具。想单独配音？用配音工具。想从头到尾全自动？一个命令搞定全流程。 未来的路线图大概是这样的： v0.1（当前）：解说漫 + 数学动画 ✅ v0.2（设计就绪）: Remotion 渲染引擎 + 素材混剪模式 v1.0：真正的 AI 视频生成（2D 动漫、3D 动画） v2.0：产品宣传片、教程视频、数据可视化等专业模板 为什么要做这个？因为我发现，做视频最难的不是“会不会剪辑软件”，而是整个流程太长了：写剧本、画分镜、找配音、剪辑合成……每一步都很耗时间。 我想做的，就是把这些步骤自动化——你只需要提供一个想法，剩下的全部交给系统。 Ruby：用 Skill 实现“一人一团队”的全栈开发 【此处可插入 Ruby 的 Growth Mate 应用实际截图】 Ruby 是我们团队的 AI 产品经理，文科生背景。 她做了一个情感陪伴 App—— Growth Mate ，专注女性成长心理领域。 但让我印象最深的不是 App 本身，而是她用 Skill 重新定义了“一个人怎么做产品”。 她不仅自己开发了整个应用，还写了一套 全自动化测试 Skill ——每天定时跑测试用例，自动生成健康度评分报告。开发、测试、部署，一个文科生用 Skill 搭建了完整的软件工程闭环。 黑客松现场，主持人对她的评价是“极具极客精神”。一个文科生，通过 Skill 实现了完整的软件工程流程—— 这本身就是“Skill 降低门槛”最好的证明。 Li Smith：用 Skill 做独立开发的“第零步” 【此处可插入 Li Smith 的市场需求雷达 Skill 实际输出报告截图】 Li Smith 是团队里的老程序员，10 年开发经验。 他的痛点很实际：想做独立开发，但不知道做什么。市面上的需求要么是红海，要么是伪需求，盲目开发容易竹篮打水。 所以他做了一个 市场需求雷达 Skill : 自动抓取 Product Hunt 等平台的新产品 → 结合自身技术栈（Node.js 等）和市场热度进行匹配打分 → 生成可行性报告和蓝海评分。 用数据驱动决策，不拍脑袋。对独立开发者来说，这可能是最实用的“第零步”Skill—— 在写第一行代码之前，先用 Skill 确认方向是对的。 其他让我印象深刻的 Skills 38 个 Skills 不可能全部介绍，挑几个我觉得特别有意思的： 🏭 环评报告自动化生成器 ——18 年资深环评人做的，200 页+的环评报告从数小时压缩到一杯咖啡的时间。效率提升 10 倍，让专家从“表格搬运工”回归决策者。 🍅 番茄采收 AI 视觉质检
